## 参与编辑

[SJTU Wiki](https://sjtu-geek.github.io/SJTU-Wiki/) 是一个由学生主导建设的校园百科平台，旨在整合上海交通大学校内各类学习、生活、事务办理等信息资源，构建便捷、高效的信息导航系统，致力于打造一个由学生主导、持续更新的校园知识库。

如果你也是校园百事通，或是对交大生活的某一方面特别精通，又或者善于总结各类信息，想将自己的知识传递给更多交大人，我们都欢迎你加入 SJTU Wiki，共同建设交大百科。

### 工作流

#### 通过 Github 贡献

要将您的文章投稿到 SJTU Wiki，您需要 Fork 一份代码仓，并在自己的代码仓中进行编辑，完成后提交 [Pull Request](https://github.com/SJTU-Geek/SJTU-Wiki/pulls) 来申请合入代码。至少两名管理员同意后，您的代码将会合入到本代码仓，并自动更新到预览网站。

#### 通过在线问卷贡献

我们还为不熟悉 GitHub 的用户准备了问卷投稿方式，您可以直接提交文本或上传 Word / Markdown 文档参与 SJTU Wiki 建设：<https://ssc.sjtu.edu.cn/f/f787ebb3>

### 内容规则

#### 立场
我们希望您能以**书面、客观的语言**描述上海交通大学的客观事实，请勿上传违反法规校纪和公序良俗的内容。

我们欢迎您**新增**对上海交通大学师生的学习生活有较强帮助的内容。我们欢迎您**修改** SJTU Wiki 上的错误内容、过时内容和表述欠佳的内容。

#### 署名
您可以在文章标题后、概述前的位置进行署名，如`作者：XXX`。请**使用恰当的昵称**。

#### 引用
若是引用了他人创作的内容，请直接在段落中或文末的参考章节中**标注**其来源，并给出对应链接。此类链接还需遵守下一节“链接”中的规定。

#### 链接
我们允许在 SJTU Wiki 上放置来自政府机构和事业单位的官方网站和文章链接。

对于非官方发布的内容、由用户自行编写的内容（如水源帖子、水源文档等），请在放出原文链接的同时在链接后标出：`<sup>[非官方内容，请谨慎参考]</sup>`。主观性较强的经验类内容也建议在首句后带上此标签。

对于外部网站推荐，请提交可靠的站点（请核查站点的可访问性、稳定性、客观性）上的链接。请注意您提交的链接有被拒绝的可能。

#### 时效性

对于时效性较强的内容，请额外提醒读者注意时效性，并给出获取最新信息的方式。SJTU Wiki 会移除过时的内容。

#### 授权
您的贡献内容以 [CC BY-NC 4.0 协议](https://creativecommons.org/licenses/by-nc/4.0/deed.zh-hans)授权给 SJTU Wiki。如需撤回授权，请通过[此问卷](https://ssc.sjtu.edu.cn/f/4282a590)联系我们。

### 编辑语法

SJTU Wiki 目前采用 GitBook 软件，将 Markdown 文件转换为一篇篇文章。

Markdown 语言非常简单易上手，如果您不了解什么是 Markdown，可以参考[这篇 Markdown 教程](https://101.lug.ustc.edu.cn/Appendix/markdown)。请特别注意，在 Markdown 中需要输入回车两次来显示换行。

请用 H1 标题（一个井号）作为页面标题，章节标题为 H2（两个井号）。页面标题到第一个段落标题之间可以编写概述。

### 架构规范

如果您在 GitHub 进行贡献，请在提交 PR 时注意以下架构规范。

#### 目录

如果您希望新增或是修改目录和文章，请记得在`.vitepress/config.mjs`的`sidebar`中进行对应更改。请将目录名和文件名命名为目录和文章的对应英文。

#### 图片

请将所有图片存放在文章所在目录的 `./images` 目录下，如 `/undergraduate-study/study-plan/category.html` 页面用到的图片请放在 `/undergraduate-study/study-plan/images` 目录中。请将图片重命名为**有意义的文件名**，使用**下划线**连接英文单词。

#### 附件

请将所有附件存放在文章所在目录的 `./files` 目录下。请将附件重命名为**有意义的文件名**。


### 积分体系
为了感谢您对 SJTU Wiki 建设做出的贡献，您的每次对 SJTU Wiki 的有效编辑都将累计积分，并可 1:1 兑换成[传承·交大](https://beta.share.dyweb.sjtu.cn/user/points/redeem)的积分。

- 积分明细可在[思源极客协会 Wiki 贡献中心](https://geek.sjtu.edu.cn/products/wiki/contrib)查询
- 贡献中心可绑定 GitHub 账号，系统自动合并您通过 GitHub 账号提交、合并的 Pull Request 对应的积分

SJTU Wiki 贡献者发放积分的总则如下：

- 新增一篇文章：约 50 分
- 新增一个章节：约 10 分
- 新增一个段落：约 1~3 分

积分细则：
- 以上文章/章节/段落的规模可参考文章《奖助学金》。可获得50积分的文章，其参考字数为 2000 字，计算时减去纯引用内容的字数。
- 修正和更新内容的分数约为新增内容的 60%。
- 内容详细程度和篇幅较突出的，可获得最多 50% 积分加成。
- 部分极小修改可能无法获得积分。
- 如果对他人的经验分享进行排版后放进 SJTU Wiki，请取得原作者授权。搬运者可获得 30% 的贡献分数，原作者可声明并领取剩余积分。
